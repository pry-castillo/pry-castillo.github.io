SPRINT BACKLOG

Es una lista de tareas identificadas por el scrum team;
ésta deberá ser completada durante cada sprint y es representado a través de un tablero de tareas
de ese modo hace visible todo el trabajo necesario para alcanzar el compromiso que se hizo
con el product owner para el sprint.
Tambien permite ver las tareas donde el equipo está teniendo problemas y no avanza para tomar decisiones al respecto,
los objetivos/requisitos para cada uno y el esfuerzo pendiente para finalizarlas.
